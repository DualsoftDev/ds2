1. 개요

DSPilot.Engine은 PLC(Programmable Logic Controller) 데이터를 수집, 분석하고 프로세스 실행 흐름을 모니터링하는 F# 라이브러리입니다. 실시간 신호 처리, 통계 분석, 병목 구간 탐지, Gantt 차트 레이아웃 등의 기능을 제공합니다.

📌 주요 특징
  • ✅ F# 순수 함수형 프로그래밍으로 안전하고 예측 가능한 코드
  • ✅ 6개 레이어로 명확하게 분리된 아키텍처
  • ✅ SQLite 기반 Unified 데이터베이스 (EV2 통합)
  • ✅ 실시간 태그 상태 추적 및 엣지 감지
  • ✅ 통계 분석 (평균, 표준편차, 변동계수, 백분위수)
  • ✅ 성능 메트릭 및 Heatmap 생성
  • ✅ Cycle 분석, 병목 구간 탐지, Critical Path 식별

2. 프로젝트 구조

DSPilot.Engine/

│   ├── Types.fs                도메인 열거형 (EdgeType, CallState, etc.)
│   ├── CallKey.fs              Call 고유 식별자
│   └── EdgeDetection.fs        신호 엣지 감지 (Rising/Falling)
│
├── Database/                  ← 데이터베이스 레이어
│   ├── Configuration.fs        DB 경로 설정 및 연결 문자열
│   ├── Entities.fs             엔티티 타입 (DspFlowEntity, DspCallEntity)
│   ├── Dtos.fs                 Dapper용 DTO
│   ├── QueryHelpers.fs         SQL 쿼리 빌더
│   ├── Repository.fs           데이터 액세스 (CRUD)
│   └── Initialization.fs       스키마 초기화 + AASX 로딩
│
├── Tracking/                  ← 상태 추적
│   ├── TagStateTracker.fs      PLC 태그 상태 추적 (Immutable)
│   └── StateTransition.fs      상태 전이 로직 (Ready→Going→Finish)
│
├── Statistics/                ← 통계 계산
│   ├── Statistics.fs           순수 통계 함수 (평균, 표준편차, CV)
│   └── RuntimeStatistics.fs    런타임 실행 통계 관리
│
├── Analysis/                  ← 분석 로직
│   ├── Performance.fs          성능 메트릭 및 색상 클래스
│   ├── CycleAnalysis.fs        시간 버킷팅, 백분위수, 분포
│   ├── GanttLayout.fs          타임라인 레인 배치 알고리즘
│   ├── BottleneckDetection.fs  병목 구간 탐지 및 Critical Path
│   └── FlowAnalysis.fs         Flow 분석, DAG, MT/WT/CT 계산
│
└── ViewModels/                ← UI 뷰모델
    └── Models.fs               Heatmap, Blueprint 뷰모델

📊 의존성 흐름

Core → Database → Tracking → Statistics → Analysis → ViewModels

각 레이어는 상위 레이어만 참조하며, 순환 참조가 없습니다.

3. Core 레이어

Core 순수 도메인 타입과 엣지 감지 로직

📄 Core/Types.fs

도메인 열거형 및 기본 타입 정의

  • EdgeType - Rising | Falling | NoChange
  • CallState - Ready | Going | Finish
  • BucketSize - Min5 | Min10 | Hour1
  • HeatmapMetric - AverageTime | StdDeviation | CoefficientOfVariation | PerformanceScore
  • ColorClass - Excellent | Good | Fair | Poor | Critical
  • TrendPoint - 시간별 트렌드 데이터 포인트
  • PerformanceMetrics - 성능 메트릭 레코드

📄 Core/CallKey.fs

Call 고유 식별자 (Struct)

  • FlowName, CallName, WorkName 복합 키
  • Create(flowName, callName) - 기본 생성
  • CreateFull(...) - Work 포함 생성
  • ToHashKey() - Dictionary 키로 사용 가능한 해시 키 생성

📄 Core/EdgeDetection.fs

신호 엣지 감지 (순수 함수)

  • detectEdge: string option -> string -> EdgeType
  • isRising: EdgeType -> bool
  • isFalling: EdgeType -> bool
  • PLC 신호의 0→1 (Rising), 1→0 (Falling) 전환 감지

4. Database 레이어

Database SQLite 기반 데이터 액세스 및 스키마 관리

📄 Database/Configuration.fs

데이터베이스 경로 설정

  • DatabasePaths - Unified 모드 DB 경로 (plc.db)
  • loadDatabasePaths - IConfiguration에서 경로 로드
  • createConnectionString - SQLite 연결 문자열 생성
  • 환경 변수 확장 지원 (%APPDATA% 등)

📄 Database/Entities.fs

엔티티 타입 정의

  • DspFlowEntity - Flow 테이블 (MT, WT, CT, State)
  • DspCallEntity - Call 테이블 (통계 필드 포함)
  • FlowState - UI 표시용 Flow 상태
  • CallStateDto - UI 표시용 Call 상태
  • DspDbSnapshot - 전체 DB 스냅샷

📄 Database/Dtos.fs

Dapper용 DTO (Option → Nullable 변환)

  • DapperFlowDto - Flow 삽입/업데이트용
  • DapperCallDto - Call 삽입/업데이트용
  • CallInfoDto - Dapper 역직렬화용
  • CallStatisticsDto - 통계 쿼리 결과용
  • FromEntity 메서드로 Entity → DTO 변환

📄 Database/QueryHelpers.fs

SQL 쿼리 빌더 헬퍼

  • dateRangeClause - 날짜 범위 WHERE 절
  • paginationClause - LIMIT/OFFSET 절
  • buildWhereClause - 동적 WHERE 절 생성
  • SQL Injection 방지를 위한 안전한 쿼리 빌딩

📄 Database/Repository.fs

데이터 액세스 레이어 (Dapper 기반)

  • getFlowsAsync - Flow 목록 조회
  • getCallsAsync - Call 목록 조회
  • upsertFlowAsync - Flow INSERT/UPDATE
  • upsertCallAsync - Call INSERT/UPDATE
  • updateCallStatisticsAsync - 통계 업데이트
  • getCallStatisticsAsync - 통계 조회
  • 비동기 Task 기반 모든 작업

📄 Database/Initialization.fs

스키마 초기화 + AASX 데이터 로딩

  • Schema.createSchemaAsync - EV2 기본 스키마 + DSP 확장 테이블 생성
  • AasxLoader.loadFromAasxAsync - AASX 파일에서 Flow/Call 로드
  • initializeDatabaseAsync - 전체 초기화 프로세스
  • Ev2Bootstrap + DspDatabaseInit 통합

5. Tracking 레이어

Tracking PLC 태그 상태 추적 및 상태 전이

📄 Tracking/TagStateTracker.fs

Immutable 태그 상태 추적

  • TagEdgeState - 태그 이름, 이전/현재 값, 엣지 타입, 갱신 시간
  • updateTagValue - 태그 값 업데이트 및 엣지 감지
  • getState - 현재 태그 상태 조회
  • 순수 함수형 스타일 (Map 기반 immutable state)
  • C# Interop을 위한 Mutable wrapper 포함

📄 Tracking/StateTransition.fs

Call 상태 전이 로직

  • Ready → Going - InTag Rising 감지 시
  • Going → Finish - OutTag Rising 감지 시
  • Finish → Ready - 다음 사이클 준비
  • shouldTransition - 상태 전이 조건 검사
  • applyTransition - 새 상태 반환 (순수 함수)

6. Statistics 레이어

Statistics 통계 계산 및 런타임 메트릭

📄 Statistics/Statistics.fs

순수 통계 함수

  • movingAverage - 이동 평균 계산
  • standardDeviation - 표준편차 계산
  • coefficientOfVariation - 변동계수 (CV = σ/μ)
  • addSample - 샘플 추가 (최대 샘플 수 제한)
  • 부작용 없는 순수 함수로 테스트 용이

📄 Statistics/RuntimeStatistics.fs

런타임 실행 통계 관리

  • RuntimeStats - 실행 시간 이력 관리
  • updateStats - 새 실행 시간 추가
  • getAverage - 평균 실행 시간
  • getStdDev - 표준편차
  • getCV - 변동계수
  • 세션/전체 카운트 추적

7. Analysis 레이어

Analysis 성능 분석, 사이클 분석, 병목 탐지

📄 Analysis/Performance.fs

성능 메트릭 및 색상 클래스

  • calculatePerformanceMetrics - 평균, 표준편차, CV 계산
  • normalizeMetrics - 0~100 정규화
  • calculatePerformanceScore - 종합 성능 점수 (가중치 적용)
  • determineColorClass - Excellent/Good/Fair/Poor/Critical 분류
  • Heatmap 시각화를 위한 색상 클래스 매핑

📄 Analysis/CycleAnalysis.fs

시간 버킷팅 및 통계 분석

  • bucketSizeToTimeSpan - 5분/10분/1시간 단위 변환
  • getBucketKey - 시간을 버킷 키로 변환
  • calculateBucketStats - 버킷별 통계 계산
  • calculatePercentiles - 50th, 95th, 99th 백분위수
  • calculateDistribution - 히스토그램 분포

📄 Analysis/GanttLayout.fs

Gantt 차트 타임라인 레인 배치

  • TimelineItem - 타임라인 아이템 (시작/종료 시간)
  • isOverlapping - 두 아이템의 시간 겹침 검사
  • assignLanes - 최소 레인 수로 아이템 배치
  • 겹치지 않는 아이템은 같은 레인에 배치
  • Gantt 차트 시각화를 위한 Y축 위치 계산

📄 Analysis/BottleneckDetection.fs

병목 구간 탐지 및 Critical Path

  • BottleneckType - CriticalPath | LongDuration | FrequentExecution
  • BottleneckInfo - 병목 정보 (Call 이름, 타입, 심각도, 추천)
  • findCriticalPath - DAG에서 가장 긴 경로 찾기
  • identifyLongDurationBottlenecks - 실행 시간이 긴 Call 식별
  • analyzeMultiCycleBottlenecks - 여러 사이클에서 반복되는 병목

📄 Analysis/FlowAnalysis.fs

Flow 분석 및 DAG 구조 분석

  • selectRepresentativeWork - 대표 Work 선택
  • buildCallDag - Call 간 의존성 DAG 구축
  • detectCycles - 순환 참조 감지
  • calculateMT - Machine Time (기계 가동 시간)
  • calculateWT - Waiting Time (대기 시간)
  • calculateCT - Cycle Time (총 사이클 시간)

10. 사용 방법

10.1 프로젝트 참조

<ItemGroup>
  <ProjectReference Include="..\..\DSPilot.Engine\DSPilot.Engine.fsproj" />
</ItemGroup>

10.2 Database 초기화

open DSPilot.Engine

// 1. Configuration 로드
let paths = DatabaseConfig.loadDatabasePaths config logger

// 2. 스키마 초기화 + AASX 로딩
let! success = DatabaseInitialization.initializeDatabaseAsync paths logger

if success then
    printfn "Database initialized successfully!"

10.3 태그 상태 추적

open DSPilot.Engine

// 태그 상태 맵 초기화
let mutable stateMap = TagStateTracker.empty

// 태그 값 업데이트
let (newMap, tagState) = TagStateTracker.updateTagValue "IN_TAG_01" "1" stateMap
stateMap <- newMap

// 엣지 검사
if EdgeDetection.isRising tagState.EdgeType then
    printfn "Rising edge detected on %s" tagState.TagName

10.4 통계 계산

open DSPilot.Engine

// 샘플 데이터
let samples = [120.0; 135.0; 125.0; 142.0; 118.0]

// 평균 계산
let avg = Statistics.movingAverage samples

// 표준편차
let stdDev = Statistics.standardDeviation samples avg

// 변동계수
let cv = Statistics.coefficientOfVariation avg stdDev

printfn "Average: %.2f, StdDev: %.2f, CV: %.2f" avg stdDev cv

10.5 성능 분석

open DSPilot.Engine

// 성능 메트릭 계산
let metrics = Performance.calculatePerformanceMetrics executionTimes

// 정규화 (0~100)
let normalized = Performance.normalizeMetrics metrics allMetrics

// 성능 점수 (가중치: CV 60%, Avg 40%)
let score = Performance.calculatePerformanceScore normalized 0.6 0.4

// 색상 클래스 결정
let colorClass = Performance.determineColorClass score

match colorClass with
| ColorClass.Excellent -> printfn "🟢 Excellent performance!"
| ColorClass.Critical -> printfn "🔴 Critical - needs attention!"
| _ -> printfn "Performance: %A" colorClass

10.6 병목 탐지

open DSPilot.Engine

// DAG 구축
let dag = FlowAnalysis.buildCallDag calls

// Critical Path 찾기
let criticalPath = BottleneckDetection.findCriticalPath dag

// 병목 구간 식별
let bottlenecks = BottleneckDetection.identifyLongDurationBottlenecks calls threshold

for bottleneck in bottlenecks do
    printfn "⚠️ Bottleneck: %s (%A)" bottleneck.CallName bottleneck.BottleneckType
    printfn "   Severity: %.2f%%" (bottleneck.Severity * 100.0)
    printfn "   Recommendation: %s" bottleneck.Recommendation

10.7 Gantt 레이아웃

open DSPilot.Engine

// 타임라인 아이템 생성
let items =
    calls
    |> List.map (fun c ->
        { Id = c.CallName
          StartTime = c.StartTime
          EndTime = c.EndTime })

// 레인 배치 (겹치지 않게)
let laneAssignments = GanttLayout.assignLanes items

// 시각화를 위한 Y축 위치 계산
for (item, lane) in laneAssignments do
    let yPos = lane * 50  // 50px per lane
    printfn "%s -> Lane %d (Y: %d)" item.Id lane yPos
