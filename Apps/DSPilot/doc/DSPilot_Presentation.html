핵심 원칙

UI는 계산 금지! 모든 계산은 DSPilot.Engine에서 사전 수행하고, UI는 Projection 테이블(dspFlow, dspCall)에서 읽기만 수행합니다.

📊 Projection 테이블 구조

테이블 용도 주요 필드
dspFlow Flow 레벨 Projection State, ActiveCallCount, MT, WT, CT, AverageCT, StdDevCT
dspCall Call 레벨 Projection State, ProgressRate, LastDurationMs, AverageGoingTime, StdDevGoingTime, FocusScore

// ❌ 금지된 코드 (UI에서 계산) var average = calls.Average(c => c.Duration); // ✅ 올바른 코드 (Projection 읽기) var average = call.AverageGoingTime;

📊 기능 1: 공정 상태 (Process Status)

목적

설비/생산 라인의 현재 운영 상태를 실시간으로 시각화 (Kanban, Default, List)

필요 Projection 필드

// dspFlow State (Ready/Going/Error) ActiveCallCount (진행 중인 Call 개수) ErrorCallCount (에러 Call 개수) // dspCall State (Ready/Going/Done/Error) ProgressRate (0.0 ~ 1.0) WorkName (Work 그룹명) Device (설비명)

계산 로직 (F#)

let updateFlowState (flowName: string) = let calls = DspRepository.getCallsByFlow flowName let activeCount = calls |> Seq.filter (fun c -> c.State = "Going") |> Seq.length let errorCount = calls |> Seq.filter (fun c -> c.State = "Error") |> Seq.length let state = if errorCount > 0 then "Error" elif activeCount > 0 then "Going" else "Ready" DspRepository.patchFlow { FlowName = flowName; State = Some state; ActiveCallCount = Some activeCount }
✅ 기대 효과:
  • 운영자가 공정 상태를 숫자보다 빠르게 이해
  • 공정 장애 시 대응 시간 단축
  • 1차 관제 역할 수행

⚡ 기능 2: 병목공정 분석 (Bottleneck Analysis)

목적

설비별 Moving Time과 Waiting Time을 비교하여 라인 내 병목 지점 탐지

필요 Projection 필드

// dspFlow MT (Moving Time) - 모든 Call의 작업 시간 합 WT (Waiting Time) - CT - MT CT (Cycle Time) - Head Start ~ Tail Finish

계산 로직 (F#)

let calculateCycleMetrics (flowName: string) = let calls = DspRepository.getCallsByFlow flowName let headCall = DspRepository.getHeadCall flowName let tailCall = DspRepository.getTailCall flowName match headCall, tailCall with | Some head, Some tail -> let mt = calls |> Seq.sumBy (fun c -> c.LastDurationMs |> Option.defaultValue 0.0) let ct = (tail.LastFinishAt - head.LastStartAt).TotalMilliseconds let wt = ct - mt DspRepository.patchFlow { FlowName = flowName; MT = Some mt; WT = Some wt; CT = Some ct } | _ -> ()
✅ 기대 효과:
  • 대기시간이 긴 병목 구간 정량 식별
  • 병목 개선 전후 비교 가능
  • 생산성 향상을 위한 설비 재배치 검토

📈 기능 3: 사이클 타임 분석 (Gantt Chart)

목적

각 설비의 동작 시퀀스를 Gantt Chart로 표현하여 타이밍과 대기 구간 분석

필요 Projection 필드

// dspCall LastStartAt (시작 시각) LastFinishAt (종료 시각) LastDurationMs (실행 시간) SequenceNo (순서) CurrentCycleNo (사이클 번호) State (상태 - 색상 구분용)

계산 로직 (F#)

let calculateGanttLayout (flowName: string) (cycleNo: int) = let calls = DspRepository.getCallsByFlow flowName |> List.filter (fun c -> c.CurrentCycleNo = cycleNo) |> List.sortBy (fun c -> c.LastStartAt) calls |> List.mapi (fun i call -> { CallName = call.CallName StartTime = call.LastStartAt EndTime = call.LastFinishAt Duration = call.LastDurationMs YPosition = i State = call.State })
✅ 기대 효과:
  • 어떤 동작이 먼저 시작하고 겹쳤는지 시각화
  • 정상 SEQ와 이상 SEQ 비교로 원인 좁히기
  • 공정 동기화 확인

🎨 기능 4: 편차 분석 (Heatmap)

목적

동작 시간의 반복 변동성을 CV(변동계수)로 수치화하여 Heatmap 시각화

필요 Projection 필드

// dspCall AverageGoingTime (평균) StdDevGoingTime (표준편차) GoingCount (실행 횟수) // UI에서 CV 계산 CV = StdDevGoingTime / AverageGoingTime

계산 로직 (F# - Welford's Method)

let updateCallStatisticsWithCV (callId: Guid) (newDuration: float) = let call = DspRepository.getCallById callId let n = call.GoingCount + 1 let oldAvg = call.AverageGoingTime |> Option.defaultValue 0.0 let oldStdDev = call.StdDevGoingTime |> Option.defaultValue 0.0 // Incremental Average let newAvg = (oldAvg * float(n - 1) + newDuration) / float(n) // Incremental StdDev (Welford's method) let oldM2 = oldStdDev * oldStdDev * float(n - 1) let delta = newDuration - oldAvg let newM2 = oldM2 + delta * (newDuration - newAvg) let newStdDev = sqrt(newM2 / float(n)) DspRepository.patchCall { CallId = callId; AverageGoingTime = Some newAvg; StdDevGoingTime = Some newStdDev; GoingCount = Some n }

색상 스케일

  • 청색 (CV < 0.1): 안정
  • 녹색 (0.1 ≤ CV < 0.3): 보통
  • 황색 (0.3 ≤ CV < 0.5): 주의
  • 적색 (CV ≥ 0.5): 불안정

📐 Incremental Statistics (Welford's Method)

왜 중요한가?

실시간 통계 계산 시 모든 과거 데이터를 다시 읽으면 성능 저하. Welford's Method로 O(1) 시간에 평균과 표준편차 계산.

Incremental Average

n = GoingCount + 1 newAvg = (oldAvg * (n - 1) + newValue) / n

Incremental Standard Deviation

// Welford's Algorithm delta = newValue - oldAvg newAvg = oldAvg + delta / n delta2 = newValue - newAvg newM2 = oldM2 + delta * delta2 newStdDev = sqrt(newM2 / n)
O(1)
계산 복잡도
< 1ms
평균 업데이트 시간
100+
초당 이벤트 처리

🎯 Focus Score (우선순위 점수)

목적

여러 Call 중 어디를 먼저 확인해야 하는지 우선순위를 자동 계산

계산 로직 (F#)

let calculate (call: DspCallEntity) (slowFlag: bool) = let mutable score = 0 // Error: +100 (최우선) if call.State = "Error" then score <- score + 100 // Unmapped: +70 (Tag 미매핑) if call.UnmappedFlag then score <- score + 70 // Slow: +50 (평균 + 2*표준편차 초과) if slowFlag then score <- score + 50 // High StdDev: +30 (변동성 큼) let stdDevRatio = match call.AverageGoingTime, call.StdDevGoingTime with | Some avg, Some stdDev when avg > 0.0 -> stdDev / avg | _ -> 0.0 if stdDevRatio > 0.3 then score <- score + 30 score

우선순위 예시

상태 점수 조치
Error + Unmapped 170 즉시 확인 필요
Error 100 긴급 조치
Unmapped 70 Tag 매핑 필요
Slow 50 성능 개선 검토
High StdDev 30 안정성 확인

🔒 핵심 설계 원칙

1. Projection 패턴

dspFlow와 dspCall은 UI용 읽기 전용 모델. UI는 계산 금지, 순수 표시만 담당.

// ❌ 금지 var average = calls.Average(c => c.Duration); // ✅ 허용 var average = call.AverageGoingTime;

2. DROP TABLE 금지

누적 통계 보존을 위해 Migration 기반 스키마 관리. ALTER TABLE만 사용.

-- ❌ 금지 DROP TABLE IF EXISTS dspFlow; -- ✅ 허용 ALTER TABLE dspFlow ADD COLUMN NewField TEXT;

3. WorkName 정확도

Call은 Work 소속이므로 work.Name을 사용해야 함 (flow.Name 아님).

// ❌ 잘못된 코드 { WorkName = flow.Name } // ✅ 올바른 코드 { WorkName = work.Name }

4. Incremental Statistics

Welford's Method로 평균, 표준편차를 O(1) 시간에 실시간 계산.

⚡ 설비 운영 효율

대기시간(WT) 분석으로 병목 제거. 평균 CT 단축 및 생산성 향상.

🎯 공정 안정성 개선

CV(변동계수) 분석으로 불안정 구간 조기 발견. 예방 정비로 고장 사전 방지.

👥 사용자 관점 가치

🛠️ 운영자 관점

  • 공정 상태: 한눈에 전체 라인 상태 파악
  • CCTV 모니터링: 신호와 실제 동작 일치 검증
  • 이상 보기: 비정상 흐름 구조적 파악

📈 공정 엔지니어 관점

  • 공정 분석: 동작별 시간 분포 확인
  • 병목공정 분석: WT > MT인 Flow 우선 개선
  • 편차 분석: CV > 0.5인 Call 안정화

🔧 유지보수 관점

  • 이력 분석: plcTagLog 필터 검색으로 원인 추적
  • 이상 리플레이: 특정 Timestamp의 Gantt + CCTV 재생
  • 이상 이력관리: 반복 이상 패턴 분석

💻 개발/자동화 엔지니어 관점

  • 시퀀스 흐름도: Prev/Next 연결 확인
  • 디지털 모델 뷰어: AASX 구조 검토
  • TAG 모니터: UnmappedFlag=1인 Call 찾아 매핑

🌟 기술 차별점

Projection 기반 아키텍처

UI는 계산 금지. 사전 계산된 dspFlow/dspCall만 읽어 실시간 성능 보장.

Incremental Statistics

Welford's Method로 평균, 표준편차를 O(1) 시간에 실시간 계산. 과거 데이터 재스캔 불필요.

CCTV 동기화

plcTagLog + Gantt + CCTV를 0.1초 단위로 동기 재생. 제어 신호 ↔ 물리 동작 검증.

다양한 시각화

Heatmap, Gantt, Diagram, Overlay, Kanban 등 목적별 최적 시각화.

누적 통계 보존

DROP TABLE 금지. Migration 기반 스키마 관리로 통계 손실 방지.

Focus Score

Error, Unmapped, Slow, High StdDev를 종합한 우선순위 자동 계산.
